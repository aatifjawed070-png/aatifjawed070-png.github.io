Lesson 1 ¬∑ Beginner ¬∑ Topic: What is the Web?

Lesson 1: What Is the Web?

The web is a network of computers that send files to each other over the internet using standard rules called protocols.[web:180][web:182][web:185][web:193][web:194]

1. Browser, Server, and HTTP

When you visit a website:[web:180][web:182][web:188][web:194]

  1. Your browser (client) sends a request asking for a web page.
  2. A web server receives the request and finds the correct files.
  3. The server sends back HTML, CSS, JavaScript, and assets.
  4. Your browser reads the HTML, fetches linked files, and renders the page.

This uses HTTP/HTTPS, which defines how requests and responses are formatted.[web:180][web:182][web:194]

2. Domain Names and IPs

Domain names like example.com map to numeric IP addresses that identify servers. DNS performs this translation for your browser.[web:180][web:185][web:194]

3. HTML, CSS, and JavaScript

  • HTML: Structure and content.[web:186][web:187][web:192]
  • CSS: Visual design and layout.[web:186][web:187][web:192]
  • JavaScript: Interactivity and behavior.[web:186][web:187][web:192]

4. Try It Yourself

Create index.html and paste:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My First Web Page</title>
</head>
<body>
  <h1>Hello, Web!</h1>
  <p>This is my first web page.</p>
</body>
</html>

Practice

  • Create a text file called notes.txt and, in your own words, describe how a browser and server talk to each other.
  • Show your index.html to a friend and explain which part is HTML, which part is the browser, and what the server would be in a real website.
Lesson 2 ¬∑ Beginner ¬∑ Topic: HTML Page Structure

Lesson 2: HTML Page Structure

Every HTML document follows a common skeleton that browsers understand.[web:168][web:170][web:186]

1. The Skeleton

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Page Title</title>
</head>
<body>
  <!-- Visible content here -->
</body>
</html>
  • <!DOCTYPE html> = HTML5 declaration.[web:168][web:170]
  • <html> wraps the entire document.[web:168][web:170]
  • <head> holds meta data, title, links to CSS/JS.[web:168][web:170]
  • <body> holds visible content.[web:168][web:170]

2. Semantic Structure

Use semantic tags to describe sections of your page.[web:167][web:169][web:187]

<body>
  <header>...</header>
  <main>
    <section>...</section>
  </main>
  <footer>...</footer>
</body>

Practice

  • Create a new page structure-test.html that uses <header>, <main>, and <footer> with any dummy text.
  • View the page in your browser and inspect it with DevTools (Elements tab) to see how the browser represents the HTML structure.

Lesson 12 ¬∑ Beginner ¬∑ Project: Todo List App

Lesson 12: Mini Project ‚Äì Todo List

In this project, you‚Äôll combine HTML, CSS, and JavaScript to build a simple todo list.[web:139][web:192]

  <h1>Todo List</h1>

  <ul id="todo-list"></ul>
</div>

2. Basic Styling (CSS)

.todo-app {
  max-width: 400px;
  margin: 0 auto;
  background: #ffffff;
  padding: 16px;
  border-radius: 10px;
  box-shadow: 0 10px 20px rgba(15, 23, 42, 0.08);
}

3. JavaScript Logic

const input = document.querySelector("#todo-input");
const addBtn = document.querySelector("#add-btn");
const list = document.querySelector("#todo-list");

addBtn.addEventListener("click", () => {
  const text = input.value.trim();
  if (!text) return;

  const li = document.createElement("li");
  li.textContent = text;

  li.addEventListener("click", () => {
    li.classList.toggle("done");
  });

  list.appendChild(li);
  input.value = "";
});

Practice

  • Extend your todo list so clicking a task twice deletes it, or add a small ‚Äúüóë‚Äù delete button for each list item.
  • Style completed tasks with a different color or a line‚Äëthrough using a .done CSS class.
